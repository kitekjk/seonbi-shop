# 🏪 전통기념품 쇼핑몰 — PRD (Product Requirements Document)

_v1.0 | 2026-03-14_

## 1. 개요

### 프로젝트명
**선비샵** (가칭) — 한국 전통/박물관 기념품 온라인 쇼핑몰

### 목적
- AI 에이전트 팀(8명)으로 실제 운영 가능한 쇼핑몰을 제작하는 데모 프로젝트
- OpenClaw 기반 "AI 팀 세팅 서비스"의 포트폴리오

### 기술 스택
- **프론트엔드:** Next.js (App Router) + Tailwind CSS
- **백엔드:** Next.js API Routes + Supabase (DB/Auth/Storage)
- **결제:** 데모용 Mock PG (실결제 X, 결제 플로우만 구현)
- **배송조회:** 데모용 Mock API (가상 배송 상태 반환)
- **배포:** Vercel
- **플랫폼:** 웹 (반응형 X, 데스크톱 우선)

---

## 2. 사용자 역할

| 역할 | 설명 |
|------|------|
| **비회원** | 상품 조회, 검색만 가능 |
| **회원** | 구매, 장바구니, 리뷰, 문의, 쿠폰 사용 |
| **관리자** | 상품/주문/배송/정산/쿠폰/CS 관리 |
| **CS 챗봇** | 상품/배송 문의 1차 자동응대 |

## 3. 기능 요구사항

### 3.1 회원 (Auth)
- [ ] 회원가입 (이메일 + 비밀번호)
- [ ] 로그인 / 로그아웃
- [ ] 마이페이지 (주문내역, 리뷰, 문의내역, 쿠폰함)
- [ ] 배송지 관리 (등록/수정/삭제, 기본배송지 설정)

### 3.2 상품
- [ ] 상품 목록 (카테고리별, 페이지네이션)
- [ ] 상품 상세 (이미지 다수, 설명, 가격, 옵션, 리뷰)
- [ ] 상품 옵션 (예: 색상, 사이즈 — 옵션별 추가금/재고)
- [ ] 상품 검색 (제목, 설명, 태그 기반 텍스트 검색)
- [ ] 상품 태그 (ex: #소주잔 #전통문양 #선물)

### 3.3 장바구니
- [ ] 상품+옵션 담기 (수량 지정)
- [ ] 수량 변경 / 삭제
- [ ] 선택 상품만 주문하기
- [ ] 쿠폰 적용 미리보기

### 3.4 주문
- [ ] 주문서 작성 (배송지, 결제수단, 쿠폰 적용)
- [ ] 결제 (데모용 Mock PG — 결제 플로우 시뮬레이션)
- [ ] 주문 상태: 결제완료 → 배송준비 → 배송중 → 배송완료
- [ ] 주문 취소 (배송준비 전까지)
- [ ] 주문 상세 조회

### 3.5 배송
- [ ] 관리자: 운송장번호 입력 → 상태 자동 변경
- [ ] 배송조회 Mock API (가상 배송 상태 시뮬레이션)
- [ ] 배송완료 시 자동 상태 업데이트

### 3.6 정산
- [ ] **구매정산:** 회원별 구매금액, 결제내역, 기간별 조회
- [ ] **판매정산:** 기간별 매출, 상품별 판매량, 수수료 차감 후 정산금액

### 3.7 쿠폰
- [ ] 상품쿠폰 (특정 상품에 적용 가능)
- [ ] 쿠폰 유형: 정액할인 / 정률할인
- [ ] 유효기간, 최소주문금액 조건
- [ ] 관리자: 쿠폰 생성/발급
- [ ] 회원: 쿠폰함에서 보유 쿠폰 확인

### 3.8 리뷰
- [ ] 상품 리뷰 작성 (텍스트 + 사진 업로드)
- [ ] 별점 (1~5)
- [ ] 리뷰 목록 (상품 상세 페이지 내)
- [ ] 본인 리뷰 수정/삭제

### 3.9 문의 & CS
- [ ] 상품 문의 (상품 상세 페이지에서 작성)
- [ ] 배송 문의 (주문 상세에서 작성)
- [ ] 일반 문의
- [ ] **CS 챗봇:** 자주 묻는 질문 자동응답 (배송 어디쯤?, 교환/반품 방법 등)
- [ ] 챗봇 → 관리자 에스컬레이션 (챗봇이 해결 못하면 관리자에게 전달)
- [ ] 관리자: 문의 목록 + 답변

### 3.10 이벤트
- [ ] 관리자: 이벤트 생성 (제목, 기간, 대표이미지, HTML 본문)
- [ ] 이벤트 목록 페이지 (진행중/종료 자동 구분)
- [ ] 이벤트 상세 페이지 (HTML 렌더링)
- [ ] 이벤트 댓글 (응모용 — 회원만 작성 가능)
- [ ] 이벤트-쿠폰 연결 (이벤트 페이지에서 쿠폰 발급)

### 3.11 관리자 페이지
- [ ] 대시보드 (오늘 주문, 매출, 미처리 문의)
- [ ] 상품 CRUD (이미지, 옵션, 태그 관리)
- [ ] 주문 관리 (상태 변경, 운송장 입력)
- [ ] 쿠폰 관리
- [ ] 문의 관리 (답변 작성)
- [ ] 정산 리포트

## 4. 데이터 모델 (주요 엔티티)

```
users          — 회원 정보
addresses      — 배송지
products       — 상품
product_options — 상품 옵션 (색상/사이즈 등)
product_images — 상품 이미지
product_tags   — 상품 태그
cart_items     — 장바구니
orders         — 주문
order_items    — 주문 상품
payments       — 결제 정보
shipments      — 배송 정보 (운송장번호, 택배사)
coupons        — 쿠폰 정의
user_coupons   — 회원 보유 쿠폰
reviews        — 리뷰 (텍스트 + 별점)
review_images  — 리뷰 이미지
inquiries      — 문의
inquiry_replies — 문의 답변
events         — 이벤트 (제목, 기간, 이미지, HTML 본문)
event_comments — 이벤트 댓글 (응모)
event_coupons  — 이벤트-쿠폰 연결
chat_logs      — 챗봇 대화 로그
```

## 5. 페이지 구조

### 고객 페이지
```
/ (홈 — 추천상품, 신상품)
/products (상품 목록)
/products/[id] (상품 상세 — 옵션선택, 리뷰, 문의)
/cart (장바구니)
/checkout (주문서)
/orders (주문 내역)
/orders/[id] (주문 상세 — 배송조회, 문의)
/mypage (마이페이지 — 리뷰, 쿠폰, 배송지)
/events (이벤트 목록)
/events/[id] (이벤트 상세)
/search?q= (검색 결과)
/login, /signup (인증)
```

### 관리자 페이지
```
/admin (대시보드)
/admin/products (상품 관리)
/admin/orders (주문 관리)
/admin/coupons (쿠폰 관리)
/admin/inquiries (문의 관리)
/admin/events (이벤트 관리)
/admin/settlement (정산)
```

## 6. 비기능 요구사항
- 웹 전용 (데스크톱 우선, 모바일 대응은 후순위)
- Supabase RLS로 데이터 접근 제어
- 이미지 저장: Supabase Storage
- SEO 기본 (메타태그, OG 태그)

## 7. MVP 범위 (Phase 1)

**포함:**
회원, 상품(옵션/태그), 장바구니, 주문, 결제(Mock), 배송(Mock), 정산, 쿠폰, 리뷰(사진), 문의(챗봇), 이벤트, 관리자, 검색

**제외 (후순위 — 운영 중 에이전트 요청으로 추가):**
배너(홈/상품 — 마케터 요청 시나리오), 모바일 앱, 소셜로그인, 위시리스트, 포인트/적립금, 교환/반품 프로세스, 다국어, 재고 알림
